JCIOTRIM WEBSITE SYSTEM REVIEW AND IMPROVEMENT LIST
Date: May 13, 2026

Project:
JCIOTRIM - Jesus Christ Is Our True Religion International Ministry

Current system summary:
- Public website is deployed on Firebase Hosting.
- Public announcement links are read from Cloud Firestore.
- Admin login uses Firebase Authentication.
- Admin approval data is stored in Cloud Firestore.
- Admin invite email uses Google Apps Script.
- Leaderboard data is fetched from Google Sheets every 5 seconds.
- Legacy local Node/Express server still exists for local fallback and older upload/admin workflows.

Overall status:
The main public website is working and the core Firebase/Google Sheets features are preserved. The strongest parts are the Firebase Hosting setup, Firestore-backed announcement links, admin-only management, and Google Sheets leaderboard refresh. The biggest improvement opportunities are security cleanup, easier announcement management, better admin audit/history, and code organization.


HIGH PRIORITY IMPROVEMENTS

1. Remove or isolate legacy hardcoded local admin credentials.
File: server.js
Why:
The legacy local server still contains default admin email, username, and password values. Firebase Hosting does not use this server, but keeping default credentials in the repository is risky and confusing.
Recommended action:
Move legacy defaults into environment variables, remove hardcoded passwords, or clearly archive the local server as development-only.

2. Fix broken text encoding on the home page.
File: church-page-main/church_HOME.html
Issue:
The Gather card currently shows a broken dash/encoding character in the phrase about "onsite or online."
Recommended action:
Replace it with normal ASCII text: "together - onsite or online."

3. Improve admin invite email result handling.
File: church-page-main/firebase-admin.js
Issue:
The invite email request uses fetch with mode: "no-cors", so the browser cannot know if the Apps Script email actually succeeded or failed. The UI may say the request was sent even if the script rejected it.
Recommended action:
Use a CORS-enabled endpoint, Firebase Cloud Functions, or Apps Script response headers that allow the site to read success/error status.

4. Add an announcement history/audit trail.
Current behavior:
The system stores only the current PDF/PPTX links in site/announcements.
Why improve:
Admins may need to know who changed links, when they changed them, and what the previous links were.
Recommended action:
Add a Firestore collection such as announcementHistory with fields:
- pdfUrl
- pptxUrl
- updatedBy
- updatedAt
- action: saved, cleared_pdf, cleared_pptx

5. Improve admin role management.
Current behavior:
Admins can approve emails and remove approved emails. Current admins are listed, but there is no polished disable/remove admin flow for existing admins.
Recommended action:
Add clear controls for:
- disable admin
- reactivate admin
- show last login/created date
- prevent an admin from removing their own access by mistake
